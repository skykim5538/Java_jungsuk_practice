# Java 기초 개념 정리

## 1. printf 포맷 지정자

### 질문: `"%5c"`에서 5는 무엇을 의미하는가?

**답변:** 5는 **필드 폭(field width)**을 의미합니다.

```java
System.out.printf("%5c", 'A');  // "    A" (총 5칸, 오른쪽 정렬)
System.out.printf("%-5c", 'A'); // "A    " (총 5칸, 왼쪽 정렬)
```

**핵심:**
- `%5c`는 총 5칸의 공간을 확보
- 기본적으로 오른쪽 정렬
- 문자는 1칸, 나머지는 공백으로 채움

## 2. 중괄호 유무에 따른 차이

### 질문: for문에서 중괄호가 있고 없고 왜 차이가 나는가?

**답변:** 중괄호 유무에 따라 for문에 포함되는 문장의 범위가 달라집니다.

```java
// 중괄호 없음 - 바로 다음 문장 1개만 포함
for (int i = 0; i < 5; i++)
    System.out.printf("%d", i);  // ← for문에 포함
    System.out.println();        // ← for문 밖! (한 번만 실행)

// 중괄호 있음 - 블록 내 모든 문장 포함
for (int i = 0; i < 5; i++) {
    System.out.printf("%d", i);  // ← for문에 포함
    System.out.println();        // ← for문에 포함! (5번 실행)
}
```

**핵심:**
- 중괄호 없으면: 바로 다음 문장 1개만 포함
- 중괄호 있으면: 블록 내 모든 문장 포함

## 3. 전위 vs 후위 연산자

### 질문: `--i`는 왜 4부터 시작하는가?

**답변:** `--i`는 전위 감소 연산자로 먼저 감소, 그 다음 값을 사용합니다.

```java
// 전위 감소 (--i)
int i = 5;
while (--i != 0) {        // 먼저 감소(5→4), 그 다음 비교(4≠0)
    System.out.println(i); // 4, 3, 2, 1 출력
}

// 후위 감소 (i--)  
int i = 5;
while (i-- != 0) {        // 먼저 비교(5≠0), 그 다음 감소(5→4)
    System.out.println(i); // 4, 3, 2, 1, 0 출력  
}
```

**핵심:**
- **전위(`--i`)**: 먼저 변경, 그 다음 사용
- **후위(`i--`)**: 먼저 사용, 그 다음 변경

## 4. do-while문의 조건

### 질문: `while (input != answer)`는 불일치하면 반복문을 종료한다는 의미인가?

**답변:** 반대입니다! **불일치하면 계속 반복, 일치하면 종료**입니다.

```java
do {
    // 실행할 코드
} while (조건);
```

- **조건이 true이면 반복 계속**
- **조건이 false이면 반복 종료**

```java
int input, answer = 50;
do {
    input = sc.nextInt();
} while (input != answer);  // input ≠ answer이면 계속, input = answer이면 종료
```

**핵심:** while 조건이 true인 동안 반복합니다.

## 5. 난수 생성의 올바른 방법

### 질문: 왜 "더 작은 수를 입력하시오"만 반복하는가?

**답변:** 난수 생성 공식이 잘못되었기 때문입니다.

```java
// 잘못된 방법 (100~199 범위)
answer = (int)(100*(Math.random()+1));

// 올바른 방법 (1~100 범위)
answer = (int)(Math.random() * 100) + 1;
```

**문제점:**
- 사용자 입력: 1~100
- 잘못된 정답: 100~199
- 결과: 입력값이 항상 정답보다 작음

**핵심:** `Math.random() * 범위 + 시작값` 공식 사용

## 6. 컴파일 오류들

### "suspicious indentation after 'if'" 의미

**답변:** if문 다음의 들여쓰기가 의심스럽다는 의미입니다.

**원인:**
- 들여쓰기 불일치
- 탭과 스페이스 혼용
- if문 다음 빈 블록

### "Unreachable statement" 의미

**답변:** 도달할 수 없는 문장이라는 뜻입니다.

**발생 경우:**
```java
return 5;
System.out.println("실행 안됨"); // ← Unreachable statement!

while(true) {
    // 무한루프
}
System.out.println("실행 안됨"); // ← Unreachable statement!
```

**핵심:** 컴파일러가 "절대 실행되지 않을 것"이라고 판단하는 코드

## 7. break문과 중첩 반복문

### 질문: 왜 두 번 체크해야 하는가?

**답변:** `break`는 가장 가까운 반복문 하나만 탈출하기 때문입니다.

```java
while (true) {                    // 바깥 반복문
    for (int x = 0; x < 10; x++) { // 안쪽 반복문
        if (sum > 100) {
            break;    // ← for문만 탈출! while문은 계속
        }
    } // ← 첫 번째 break는 여기로 이동
    
    if (sum > 100) {
        break;        // ← while문 탈출
    }
} // ← 두 번째 break는 여기로 이동
```

**핵심:**
- break는 가장 가까운 루프만 탈출
- 중첩된 반복문 개수만큼 break 필요
- `if(조건) break`는 하나의 세트처럼 사용

### while(true)와 break의 관계

**while(true)**: "영원히 반복하라"는 규칙  
**break**: "그 규칙을 무시하고 즉시 탈출하라"는 명령

```java
while (true) {      // ← 원래는 무한반복
    // 코드들
    
    if (조건) {
        break;      // ← "규칙을 무시하고 강제로 루프 탈출!"
    }
}
// break 덕분에 여기 도달 가능!
```

## 8. 핵심 요약

1. **printf 포맷**: `%숫자c`는 총 칸수를 의미
2. **중괄호**: 없으면 다음 문장 1개만, 있으면 블록 전체
3. **증감 연산자**: 전위는 먼저 변경, 후위는 나중에 변경
4. **do-while**: 조건이 true인 동안 반복
5. **난수**: `Math.random() * 범위 + 시작값`
6. **break**: 가장 가까운 루프만 탈출
7. **무한루프**: `while(true)` + `if(조건) break` 패턴 자주 사용
